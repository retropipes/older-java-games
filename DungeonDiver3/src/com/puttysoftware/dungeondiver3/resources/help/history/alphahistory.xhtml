DungeonDiver3 Alpha Version History

 

V1.0.0-alpha4 (04/06/2012):

New Features:

  • •Added a new menu item to the Help menu: What's New in DungeonDiver3. Invoking this displays the game's version history. 

  • •Added a new preference to the Misc. tab: Are Character Changes Permanent; the default is no. If this is enabled, character status changes are written back to the corresponding character files. 

Bug Fixes:

  • •Fixed a bug that caused members of your party to not display before their first move in battles after the first. 

  • •Fixed a bug that caused the game to fail to realize that a battle is in progress. 

  • •Fixed two bugs with the Prestige Viewer: 

    • ◦Clicking the View Prestige... command in the Game menu previously did nothing. Now it shows the Prestige dialog like it is supposed to. 

    • ◦The colors for the various elements of prestige (which couldn't be seen due to the first bug) were wrong. The colors have now been corrected. 

  • •Fixed a color bug with the panels in the various stat displays. 

Other Changes:

  • •Improved integration with the operating system. 

  • •Halved the number of action points consumed by stealing and draining in battle. 

 

V1.0.0-alpha3 (04/04/2012):

Bug Fixes:

  • •Fixed a freezing bug that occurred intermittently while the game was playing sounds. 

Other Changes:

  • •Increased the average number of battles required to gain a level, in turn increasing the amount of gold earned before leveling up. 

  • •Doubled the maximum amount of gold possible to earn per battle. 

  • •Reduced the maximum level attainable from 250 to 99. 

  • •Changed the formula for computing the price of enhancements to weapons and armor – it now increases with the power of the equipment being enhanced. 

  • •Changed the formula for the base price of weapons and armor to make gear a LOT cheaper as level increases, as an incentive to purchase it. 

  • •Changed the formula for the number of battles required to gain a level to be more reasonable as level increases. 

    • ◦As a comparison example, to get from level 98 to 99 under the old system would have required over 1,000,000,000,000 (1 quadrillion) battles. 

    • ◦The new, more reasonable formula requires 200 battles to get from level 98 to level 99. 

 

V1.0.0-alpha2 (04/02/2012):

Bug Fixes:

  • •Fixed menu bugs: 

    • ◦It is no longer possible to save your game while in the middle of battle. 

    • ◦The game no longer prompts you to save if you are in the middle of battle and quit the game. 

    • ◦Registering and unregistering characters can only be done while NOT playing a game, now. 

  • •Fixed faith data bug causing all effects that do damage to not work. 

  • •Fixed poisoning to work more reliably and consistently. 

  • •Fixed several issues with the updater. 

Other Changes:

  • •Changed the scoring system slightly: 

    • ◦Score tables are no longer saved or used, since only the first entry is ever populated. 

    • ◦You are no longer prompted for a name for the score table; instead, your final score is shown in a dialog. 

    • ◦The View Score Table... command in the Game menu was removed.
